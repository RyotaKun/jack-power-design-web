Tokens · 01

The atoms of design.

A design token is a named entity that stores a single design decision — a color, a spacing value, a radius. Author once in JSON, transform to every platform. Salesforce coined the term in 2016. The W3C ratified the spec in 2025.

01 · Author
One JSON file
{
  "color.accent": {
    "$value": "#2DD4BF",
    "$type": "color"
  }
}
DTCG-compliant tokens — portable across Figma, code, iOS, Android.
→
02 · Transform
CSS custom properties
:root {
  --color-accent: #2DD4BF;
  --space-4: 16px;
  --radius-md: 8px;
}
Style Dictionary or Tokens Studio emits CSS, JS, iOS, Android — same source.
→
03 · Apply
Every component inherits
.btn-primary {
  background: var(--color-accent);
  padding: var(--space-4);
}
2016
Salesforce coins the term. Jina Anne & Jon Levine introduce design tokens in the Lightning Design System — visual decisions abstracted from any single platform. designtokens.org →
2025
W3C ratifies the spec. The Design Tokens Community Group publishes v1 — JSON interchange format with $value, $type, $description. W3C DTCG →
shadcn/ui · 02

Not a library. Your library.

In 2023, shadcn (Hunter Reaves) shipped something stranger than a component library — a CLI that pastes the actual source code into your repo. You own it. You can fork it. There's no version to upgrade against. It killed the npm-component category.

The old waynpm i mui

Component library

Install an opaque dependency. Customize via theme overrides, prop drilling, !important hacks. Locked to the maintainer's pace and design choices.
~/project
$ npm i @mui/material
# 247 packages installed
$ npm ls
# node_modules: 18,432 files
  • Style overrides fight the library
  • Major versions break your UI
  • Can't fork — it's a dependency
The new wayshadcn add

Copy-paste source

A CLI writes the component file into components/ui/. Source code lives in your repo. Edit it freely. No npm dependency. No upgrade path. You own it.
~/project
$ npx shadcn@latest add button
# ✓ Created components/ui/button.tsx
$ cat components/ui/button.tsx
export const Button = ...
  • The code is yours — edit it
  • No version drift, no breaking changes
  • Built on Radix + Tailwind underneath
"This is not a component library.
It is how you build your component library."
— ui.shadcn.com · the docs, opening line
Tailwind v4 · 03

Utility-first, token-fluent.

Adam Wathan released Tailwind on Halloween 2017. Utility-first replaced semantic CSS because semantic class names were the reason CSS was hard to maintain. v4 went CSS-first, OKLCH-default, and turned the config file into part of your design system.

The config is the token file.

Tailwind v4 (January 2025) moved configuration into CSS itself — @theme blocks expose every token as a CSS variable, no JavaScript build step required.

Each utility name is a token reference. bg-sky-500 isn't a style — it's a binding to --color-sky-500 in the theme.

  • CSS-first config · @theme {} replaces tailwind.config.js
  • OKLCH palette by default — wider P3 gamut
  • Every token = a CSS variable, runtime-themable
  • 50× faster builds via Lightning CSS
/* @theme exposes tokens as CSS vars */
@theme {
  --color-accent: oklch(0.78 0.13 195);
  --radius-md: 0.5rem;
}

<!-- utility = token reference -->
<button class="bg-accent rounded-md px-4 py-2">
  Save
</button>

Color · 05

12 steps. OKLCH. Auto dark mode.

A modern color scale has 12 steps — each step has a job. 1-2 are app backgrounds. 9 is the solid action color. 11-12 are text. Flip a class to dark and the whole scale swaps in place. OKLCH replaced HSL because adding 10% lightness now actually looks 10% lighter, across every hue.

Each step has a job — backgrounds, borders, interactive states, text. The 12-step model means you stop inventing colors mid-design.
HSL · the legacy

Uneven steps

Equal lightness deltas in HSL produce wildly different perceived brightness across hues. Yellow at L=50% looks lighter than blue at L=50%.

OKLCH · the upgrade

Perceptually even

Built on Björn Ottosson's Oklab (2020). Lightness is uniform across hues. Wider P3 gamut. Tailwind v4 default, Vercel Geist default.

Grid + type · 06

8 points. One ratio.

Two rules that quietly cover most of "this looks designed." Every spacing value is a multiple of 8. Every type size derives from one modular ratio. No 13px. No 27px. No ad-hoc.

Bryn Jackson · 2015

The 8-point grid

Spacing values from a single scale: 8 · 16 · 24 · 32 · 48 · 64 · 96. Divides cleanly across 1x, 1.5x, 2x, 3x density screens. Material's 8dp baseline followed.

Multiples of 8 — pick from this set, never anything else.
8 · 16 · 24 · 32 · 48 · 64 · 96 · 128
Modular scale

One ratio, all sizes

Base 16px times a single ratio — Major Third (1.25), Perfect Fourth (1.333), Golden Ratio (1.618). Every heading on the page derives from one number. Tailwind defaults to 1.25.

Motion + theming · 08

The final layer.

Motion gives a system its personality. Theming gives it adaptability. Two layers that turn a static page into something that feels alive — and respects the visitor's device.

Motion · Matt Perry

Curves with meaning

Motion (formerly Framer Motion) standardized declarative animation in React. Material 3 codified the four canonical curves — each one fits a different intent.
